# Storage Requirements

## Purpose

Storage 模块定义文件对象存储、文件元数据、文件内容读取、对象引用和分片上传需求。

Storage 负责“文件作为可复用资源如何被保存、读取和引用”，不负责具体业务内容规则。业务模块只保存稳定文件对象标识或自身语义包装后的引用，不保存底层对象键、物理路径或存储实现细节。

## Scope

覆盖：
- 后台普通文件上传。
- 大文件分片上传。
- 文件对象查询。
- 文件内容读取。
- 文件对象删除。
- 文件对象引用建立和清理。
- 本地文件和 S3 兼容对象存储适配。
- 文件对象状态和引用状态维护。

不覆盖：
- 通用 portal 上传入口。
- CDN 分发。
- 图片裁剪、压缩和水印。
- 视频转码。
- 音频处理。
- 在线预览转换。
- 文件安全扫描。
- 邮件通知。

## Functional Requirements

- 必须支持后台上传文件。
- 上传请求必须支持 `multipart/form-data`。
- 必须校验文件是否为空。
- 必须校验文件大小限制。
- 必须校验文件后缀或内容类型是否允许。
- 上传成功后必须保存文件内容。
- 上传成功后必须生成文件对象记录。
- 上传响应必须返回稳定文件对象标识、原始文件名、内容类型、文件大小和内容读取地址。
- 必须支持按文件对象标识读取文件内容。
- 文件内容读取必须输出正确 `Content-Type`。
- 必须支持按文件名、内容类型、上传人、对象状态和引用状态查询文件对象。
- 必须支持业务模块建立文件对象引用。
- 必须支持业务模块清理文件对象引用。
- 必须支持删除文件对象。
- 已删除文件对象不得继续读取、引用或用于业务绑定。
- 必须支持初始化分片上传。
- 必须支持上传分片。
- 必须支持完成分片上传。
- 必须支持取消分片上传。
- 分片上传完成后必须生成普通文件对象。

## Business Rules

- 文件二进制内容不得进入业务数据库主数据。
- 业务模块不得保存底层对象键、物理路径或存储桶实现细节。
- 同一业务对象重复建立同一文件引用应保持幂等。
- 文件对象存在引用时，不得被普通业务流程误判为可直接清理。
- 清理最后一个引用后，文件对象必须回到未引用状态。
- 删除文件对象必须同步释放或标记底层存储内容。
- 文件内容读取不得要求调用方感知底层存储类型。
- 通用上传入口只面向后台；portal 需要上传时必须由具体业务模块定义专用上传入口并复用 Storage。
- 业务专用上传入口不得绕过 Storage 写入文件内容。
- 本地文件和 S3 兼容存储只能作为运行时底层存储实现，不改变业务语义。

## Acceptance Criteria

- 管理员能在后台上传文件并获得文件对象标识。
- 空文件、超限文件和不允许类型文件会被拒绝。
- 已上传文件能通过内容读取地址稳定读取。
- 业务模块能建立和清理文件对象引用。
- 有引用的文件对象不会被误清理。
- 删除后的文件对象不能继续读取。
- 大文件能通过分片上传完成并生成文件对象。
- 取消分片上传后，未完成内容不会生成可用文件对象。
