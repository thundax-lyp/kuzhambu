# Wangqi Document Requirements

## Purpose

王圻文档知识库模块定义王圻相关文档的数字化管理、阅读和整理需求。

## Scope

覆盖：
- 文档查看、创建、编辑、删除。
- 文档原始文件关联和替换。
- 文档全文、摘要、标签和问答对管理。
- 列表浏览、搜索和时间线浏览。
- AI 摘要、标签和问答对生成入口及候选结果确认。
- 单文档追加式问答入口。
- 文档公开和私有可见性管理。
- 版本历史、版本对比和历史恢复。
- CSV、JSON 和 HTML 设定集导出。

不覆盖：
- 三才图会门类卷导航。
- 明代习俗标签云。
- AI 摘要、标签和问答对的模型调用执行。
- 通用公开文件上传服务。
- 分享链接生成和分享访问控制。
- 三才图会静态展示页面生成。


## Functional Requirements

- 必须支持文档查看、创建、编辑和删除。
- 必须支持文档原始文件关联和替换。
- 必须支持文档全文阅读。
- 必须支持文档内容安全展示。
- 必须支持摘要、标签和问答对展示。
- 必须支持在文档编辑页或详情页内联查看、编辑和保存摘要。
- 必须支持在文档编辑页或详情页内联查看、编辑和保存标签。
- 必须支持在文档编辑页或详情页内联查看、编辑、删除和新增问答对。
- 必须支持从文档编辑页或详情页触发 AI 摘要生成、标签提取和问答对生成。
- 必须支持 AI 生成结果在候选结果区预览、修改、确认和放弃。
- 必须支持文档搜索和时间线浏览。
- 必须支持列表中文档标题、标签预览、摘要预览和时间信息展示。
- 必须支持对多选文档批量修改公开或私有状态。
- 必须支持从文档详情发起单文档问答。
- 必须支持版本历史、版本对比和历史恢复。
- 必须支持按筛选结果或选中文档导出 CSV、JSON 和 HTML 设定集。
- 必须支持用户查看、下载和删除自己的导出记录。
- 必须展示导出产物的生成时间、文档数量、过期状态和内容变更提示。

## Business Rules

- 文件替换不得破坏文档正文。
- AI 生成摘要、标签和问答对必须先确认再应用。
- 用户放弃 AI 候选结果时不得修改文档已有内容。
- 摘要和问答对属于文档上下文内的精修能力，不要求用户进入独立数据精修工作台。
- 标签编辑属于文档上下文内的内容维护能力，同时受统一标签体系治理。
- 单文档追加式问答只能围绕当前文档内容组织上下文，不替代跨知识库智能问答。
- 时间线浏览按文档时间信息排序；文档没有时间信息时按审计记录中的创建操作时间排序。
- 文档权限遵循平台统一权限规则。
- 王圻文档通过公开或私有控制平台内可见性，不要求草稿、发布或归档生命周期。
- 私有文档不得出现在非授权用户的列表、搜索、问答候选和默认导出结果中。
- 批量修改公开或私有状态必须复用单条状态修改规则，并遵守当前用户权限。
- 批量修改公开或私有状态必须展示成功数、失败数和失败原因，单条失败不得影响其他文档。
- 用户取消批量修改公开或私有状态后，已完成结果应保留。
- 删除文档前必须二次确认。
- 导出前必须按权限过滤文档。
- 导出包含私有文档时必须二次确认。
- 过期导出记录不可继续下载。
- 导出产物生成后，若导出范围内文档发生变更，导出记录必须提示“内容可能已变更”；该提示不等同于过期，未过期产物仍可下载。
- HTML 设定集导出模板是随系统发布的静态资源，不属于提示词模块或用户上传内容。
- HTML 设定集模板必须支持数据集元信息、目录、文档内容、离线打开、浏览器打印和 PDF 生成。
- 文件缺失时文档正文仍应可阅读。

## Acceptance Criteria

- 用户能新增文档并维护正文。
- 用户能关联或替换文档原始文件。
- 用户能在文档详情中查看正文、摘要、标签和问答对。
- 用户能触发摘要、标签和问答对生成，并在确认后应用。
- 用户能从文档详情发起围绕当前文档的问答。
- 用户能在文档上下文中维护摘要、标签和问答对。
- 用户能在列表浏览和时间线浏览之间切换。
- 用户能查看和恢复历史版本。
- 用户能多选文档并批量修改公开或私有状态。
- 批量修改公开或私有状态完成后，用户能看到成功数、失败数和失败原因。
- 非授权用户无法在列表、搜索、单文档问答或默认导出中使用私有文档。
- 用户能选择王圻文档范围和格式后生成导出产物。
- 用户无权导出的文档不会进入导出结果。
- 过期导出记录不可继续下载。
