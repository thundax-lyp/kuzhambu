# Sharing Requirements

## Purpose

分享模块定义跨知识库内容的在线只读分享需求。

## Scope

覆盖：
- 三才图会、王圻文档和明代习俗内容分享。
- 单链接多内容分享。
- 公开分享和私有分享。
- 分享撤销和恢复。
- 分享过期。
- 只读访问页。
- 访问统计。
- 私有内容公开分享风险提示。

不覆盖：
- 静态 HTML 展示页面。
- 数据导出文件。
- CDN 分发。
- 内容编辑。


## Functional Requirements

- 必须支持跨库选择内容生成分享链接。
- 单个分享链接必须支持多个内容。
- 必须支持批量创建分享链接。
- 分享链接必须支持公开或私有。
- 分享链接必须支持过期时间。
- 分享链接必须支持撤销和恢复。
- 分享访问页必须只读展示。
- 必须支持访问统计。

## Business Rules

- 分享链接公开状态与内容平台内私有状态相互独立。
- 批量创建分享链接时，每条分享链接仍必须独立设置公开或私有。
- 批量创建分享链接必须展示成功数、失败数和失败原因，单条失败不得影响其他内容。
- 用户取消批量创建分享链接后，已创建成功的分享链接应保留。
- 公开分享私有内容必须二次确认，确认文案固定为："你正在公开分享平台内私有内容。创建后，获得链接的人无需登录即可查看这些内容。此操作不会改变内容在平台内的私有状态，但会绕过平台内列表、搜索和问答的可见性限制。是否继续？"
- 私有内容创建私有分享时，提示文案固定为："该分享包含平台内私有内容。私有分享仅创建者和管理员可访问，内容在平台内的私有状态不变。"
- 私有分享仅创建者和管理员可访问。
- 撤销或过期后链接不可访问。
- 分享目标被删除时显示占位，其余目标正常展示。

## Acceptance Criteria

- 用户能创建分享链接并复制。
- 用户能对多选内容批量创建分享链接。
- 批量创建分享链接完成后，用户能看到成功数、失败数和失败原因。
- 被撤销或过期的链接不可访问。
- 分享目标被删除时访问页显示占位提示。
- 私有分享未登录时引导登录。
