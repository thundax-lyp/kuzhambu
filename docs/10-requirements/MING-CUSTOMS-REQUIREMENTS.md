# Ming Customs Requirements

## Purpose

明代习俗知识库模块定义明代社会习俗专题内容的管理、筛选和阅读需求。

## Scope

覆盖：
- 习俗条目查看、创建、编辑、删除。
- 概述、正文、分类、关键词、标签和原文摘录展示。
- Markdown 内容安全渲染。
- 习俗列表、关键词搜索和详情弹窗。
- 标签云筛选。
- 摘要、标签和问答对展示。
- 公开和私有可见性管理。
- 版本历史、版本对比和历史恢复。
- CSV、JSON 和 HTML 设定集导出。

不覆盖：
- 三才图会门类卷结构。
- 王圻文档文件替换和时间线。
- AI 生成的执行。
- 三才图会静态展示页面生成。
- 草稿、发布和归档生命周期。


## Functional Requirements

- 必须支持习俗条目基础 CRUD。
- 必须支持习俗概述、正文、分类、关键词、标签和原文摘录展示。
- 必须支持习俗列表浏览和关键词搜索。
- 必须支持习俗详情弹窗。
- 必须支持 Markdown 内容安全渲染。
- 必须支持标签云筛选。
- 必须支持对多选习俗条目批量修改公开或私有状态。
- 必须支持公开和私有可见性管理。
- 必须支持摘要、标签和问答对展示。
- 必须支持在习俗编辑页或详情页内联查看、编辑和保存摘要。
- 必须支持在习俗编辑页或详情页内联查看、编辑、删除和新增问答对。
- 必须支持版本历史、版本对比和历史恢复。
- 必须支持按分类、标签、筛选结果或选中条目导出 CSV、JSON 和 HTML 设定集。
- 必须支持用户查看、下载和删除自己的导出记录。
- 必须展示导出产物的生成时间、条目数量和过期状态。

## Business Rules

- Markdown 必须安全渲染，禁止危险 HTML。
- 习俗详情弹窗必须展示标题、概述、正文、标签和原文摘录。
- 标签云必须按标签使用频率体现强弱，点击标签后筛选习俗列表。
- 标签云筛选必须遵守权限过滤。
- 私有习俗不得进入未授权用户可见的列表、搜索结果、标签云统计、问答候选、分享默认选择或导出默认范围。
- 明代习俗不使用草稿、发布和归档生命周期；可见性由公开或私有控制。
- 批量修改公开或私有状态必须复用单条状态修改规则，并遵守当前用户权限。
- 批量修改公开或私有状态必须展示成功数、失败数和失败原因，单条失败不得影响其他习俗条目。
- 用户取消批量修改公开或私有状态后，已完成结果应保留。
- 删除习俗条目前必须二次确认。
- AI 生成结果必须先确认再应用。
- 摘要和问答对属于习俗上下文内的精修能力，不要求用户进入独立数据精修工作台。
- 导出前必须按权限过滤习俗条目。
- 导出包含私有习俗条目时必须二次确认。
- 过期导出记录不可继续下载。
- HTML 设定集导出模板是随系统发布的静态资源，不属于提示词模块或用户上传内容。
- HTML 设定集模板必须支持数据集元信息、目录、习俗内容、Markdown 安全渲染结果、离线打开、浏览器打印和 PDF 生成。
- 无标签内容仍可保存。

## Acceptance Criteria

- 用户能按标签筛选习俗条目。
- 用户能通过关键词搜索习俗条目。
- 用户能打开习俗详情并阅读安全渲染后的内容。
- 用户无权访问的私有习俗不会出现在列表、搜索和标签云中。
- 用户能在习俗上下文中维护摘要和问答对。
- 用户能维护习俗内容并查看历史版本。
- 用户能多选习俗条目并批量修改公开或私有状态。
- 批量修改公开或私有状态完成后，用户能看到成功数、失败数和失败原因。
- 用户能选择明代习俗范围和格式后生成导出产物。
- 用户无权导出的习俗条目不会进入导出结果。
- 过期导出记录不可继续下载。
