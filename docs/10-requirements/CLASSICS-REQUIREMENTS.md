# Classics Requirements

## Purpose

Classics 古籍域承载平台三类古籍知识库内容和跨知识库在线分享能力。

本文档按原始需求边界组织，而不是按后台页面区域重新划分：

- 三才图会知识库。
- 王圻文档知识库。
- 明代习俗知识库。
- 跨知识库分享。

## Scope

覆盖：

- 三才图会 14 个正式门类、106 卷、卷首辅助内容和条目的三级浏览、整理、维护、导出、静态展示页面和视觉资产生产。
- 王圻相关文档的创建、编辑、阅读、原始文件关联、时间线、单文档问答入口、导出和版本管理。
- 明代习俗专题条目的创建、编辑、筛选、阅读、标签云、导出和版本管理。
- 三才图会、王圻文档和明代习俗的摘要、标签和问答对在内容上下文内联维护。
- 三才图会、王圻文档和明代习俗的公开和私有可见性管理。
- 跨知识库单链接多内容分享、批量创建分享链接、公开分享、私有分享、撤销、恢复、过期、只读访问页和访问统计。

不覆盖：

- AI 服务、模型和提示词配置。
- AI 翻译、标签提取、摘要生成、问答对生成、图片理解和条目拆分的底层执行。
- 跨库搜索排序、查询理解和搜索日志。
- 跨库智能问答会话和回答生成。
- 知识图谱、实体关系抽取和数据精修工作台。
- 通用文件对象存储、CDN 分发、图片裁剪、图片压缩、水印、视频和音频处理。

## Functional Requirements

### 三才图会知识库

- 必须支持按门类、卷、条目逐级浏览。
- 必须支持 14 个正式门类、106 卷和卷首辅助内容浏览。
- 必须支持门类和卷的稳定排序。
- 必须支持条目查看、创建、编辑和删除。
- 必须支持编辑条目标题、门类、卷、原文、译文和标签。
- 必须展示条目原文、译文、标签、配图和状态。
- 必须支持多张配图展示、缩略预览和放大浏览。
- 必须区分原始配图和视觉资产工作流生成图。
- 必须支持从条目上下文进入该条目的视觉资产工作流。
- 必须支持原图上传、删除和预览。
- 必须支持基于图片理解结果进行信息融合。
- 必须支持文本和图片理解权重调节。
- 必须支持视觉描述生成、编辑和预览。
- 必须支持 AI 生图入口。
- 必须支持视觉资产产物历史和当前使用版本选择。
- 必须支持对多选条目批量执行视觉资产相关处理。
- 必须支持在条目编辑页或详情页内联查看、编辑和保存摘要。
- 必须支持在条目编辑页或详情页内联查看、编辑、删除和新增问答对。
- 必须支持分页、筛选、当前卷内搜索和多选。
- 分页必须至少支持每页 50、100 和 200 条。
- 必须支持按翻译状态、配图状态、视觉资产状态、完善状态和公开状态筛选。
- 必须支持对多选条目批量修改公开或私有状态。
- 必须支持自动保存、手动保存和草稿恢复提示。
- 必须支持版本历史、版本对比和历史恢复。
- 必须支持草稿、发布和归档生命周期管理。
- 必须支持公开和私有可见性管理。
- 必须支持按门类、卷、筛选结果或选中条目导出 CSV、JSON 和 HTML 设定集。
- 必须支持按条目、筛选结果或选中视觉资产导出 HTML 视觉资产设定集。
- 必须支持后台用户按权限查看、下载和删除导出记录。
- 必须展示导出产物的生成时间、条目数量、资产数量和过期状态。
- 必须支持三才图会静态展示页面生成。
- 静态展示页面必须支持门类概览、卷列表、条目详情、搜索、筛选、浏览模式切换和响应式布局。
- 静态展示页面门类概览必须展示门类名称、条目总数、配图条目数和代表性缩略图。
- 静态展示页面条目详情必须展示条目名称、原文、译文、标签、配图和视觉资产描述。

### 王圻文档知识库

- 必须支持文档查看、创建、编辑和删除。
- 必须支持文档原始文件关联和替换。
- 必须支持文档全文阅读和内容安全展示。
- 必须支持摘要、标签和问答对展示。
- 必须支持在文档编辑页或详情页内联查看、编辑和保存摘要。
- 必须支持在文档编辑页或详情页内联查看、编辑和保存标签。
- 必须支持在文档编辑页或详情页内联查看、编辑、删除和新增问答对。
- 必须支持从文档编辑页或详情页触发 AI 摘要生成、标签提取和问答对生成。
- 必须支持 AI 生成结果在候选结果区预览、修改、确认和放弃。
- 必须支持文档搜索和时间线浏览。
- 必须支持列表中文档标题、标签预览、摘要预览和时间信息展示。
- 必须支持对多选文档批量修改公开或私有状态。
- 必须支持从文档详情发起单文档问答。
- 必须支持版本历史、版本对比和历史恢复。
- 必须支持按筛选结果或选中文档导出 CSV、JSON 和 HTML 设定集。
- 必须支持后台用户按权限查看、下载和删除导出记录。
- 必须展示导出产物的生成时间、文档数量和过期状态。

### 明代习俗知识库

- 必须支持习俗条目查看、创建、编辑和删除。
- 必须支持习俗概述、正文、分类、关键词、标签和原文摘录展示。
- 必须支持习俗列表浏览和关键词搜索。
- 必须支持习俗详情弹窗。
- 必须支持 Markdown 内容安全渲染。
- 必须支持标签云筛选。
- 必须支持对多选习俗条目批量修改公开或私有状态。
- 必须支持公开和私有可见性管理。
- 必须支持摘要、标签和问答对展示。
- 必须支持在习俗编辑页或详情页内联查看、编辑和保存摘要。
- 必须支持在习俗编辑页或详情页内联查看、编辑、删除和新增问答对。
- 必须支持版本历史、版本对比和历史恢复。
- 必须支持按分类、标签、筛选结果或选中条目导出 CSV、JSON 和 HTML 设定集。
- 必须支持后台用户按权限查看、下载和删除导出记录。
- 必须展示导出产物的生成时间、条目数量和过期状态。

### 分享

- 必须支持选择三才图会、王圻文档和明代习俗内容生成分享链接。
- 单个分享链接必须支持多个内容。
- 必须支持对多选内容批量创建分享链接。
- 分享链接必须支持公开或私有。
- 分享链接必须支持过期时间。
- 分享链接必须支持撤销和恢复。
- 分享访问页必须只读展示。
- 必须支持访问统计。

## Business Rules

### 三才图会规则

- 自动保存用于防丢，不等同于正式版本。
- 手动保存、AI 结果应用和历史恢复应产生正式版本。
- 摘要和问答对属于内容上下文内的精修能力，不要求用户进入独立数据精修工作台。
- 私有条目只对授权用户可见。
- 新建条目应先进入草稿状态。
- 已发布条目不可退回草稿；需要退出默认可见范围时应归档。
- 草稿、发布和归档生命周期与公开或私有可见性相互独立。
- 归档条目不参与默认列表、搜索、问答、导出和静态展示。
- 归档条目可在知识图谱中保留来源标记。
- 删除条目前必须二次确认。
- 条目删除后，关联分享目标应显示内容已删除的占位提示，不应导致分享页崩溃。
- 条目拆分后保存的子条目是独立条目；父条目被删除时不应自动删除已保存子条目。
- 图片缺失不得影响正文阅读，必须显示占位提示。
- 视觉资产必须关联明确三才图会条目。
- 原图和 AI 生成图必须可区分。
- 生图失败不得影响已有原图和历史产物。
- 删除原图或视觉资产产物前必须二次确认。
- 批量视觉资产处理必须复用单条视觉资产流程规则，单条失败不影响其他条目。
- 用户取消批量视觉资产处理后，已完成结果应保留。
- 视觉资产设定集必须包含条目来源、视觉描述、色彩或风格信息、原图和 AI 生成图引用。
- 视觉资产设定集导出前必须按权限过滤关联条目和视觉资产。
- 用户可以选择当前使用版本。
- 三才图会静态展示页面模板是随系统发布的静态资源，必须与 HTML 设定集模板分离。
- 静态展示页面默认只包含公开且未归档条目。
- 静态展示页面包含私有条目时必须由管理员确认，确认文案固定为："你正在将平台内私有内容写入静态展示页面。生成后的 HTML 文件将脱离平台登录、权限、撤销和搜索可见性控制；任何获得文件或访问地址的人都可能查看这些内容。此操作不会改变内容在平台内的私有状态。是否继续生成？"
- 静态展示页面一旦生成即脱离平台权限、撤销和内容状态变更自动控制。

### 王圻文档规则

- 文件替换不得破坏文档正文。
- AI 生成摘要、标签和问答对必须先确认再应用。
- 用户放弃 AI 候选结果时不得修改文档已有内容。
- 摘要和问答对属于文档上下文内的精修能力，不要求用户进入独立数据精修工作台。
- 标签编辑属于文档上下文内的内容维护能力，同时受统一标签体系治理。
- 单文档追加式问答只能围绕当前文档内容组织上下文，不替代跨知识库智能问答。
- 时间线浏览按文档时间信息排序；文档没有时间信息时按创建时间排序。
- 王圻文档通过公开或私有控制平台内可见性，不要求草稿、发布或归档生命周期。
- 私有文档不得出现在非授权用户的列表、搜索、问答候选和默认导出结果中。
- 文件缺失时文档正文仍应可阅读。

### 明代习俗规则

- Markdown 必须安全渲染，禁止危险 HTML。
- 习俗详情弹窗必须展示标题、概述、正文、标签和原文摘录。
- 标签云必须按标签使用频率体现强弱，点击标签后筛选习俗列表。
- 标签云筛选必须遵守权限过滤。
- 私有习俗不得进入未授权用户可见的列表、搜索结果、标签云统计、问答候选、分享默认选择或导出默认范围。
- 明代习俗不使用草稿、发布和归档生命周期；可见性由公开或私有控制。
- AI 生成结果必须先确认再应用。
- 摘要和问答对属于习俗上下文内的精修能力，不要求用户进入独立数据精修工作台。
- 无标签内容仍可保存。

### 通用内容规则

- 内容权限遵循平台统一权限规则。
- 批量修改公开或私有状态必须复用单条状态修改规则，并遵守当前用户权限。
- 批量修改公开或私有状态必须展示成功数、失败数和失败原因，单条失败不得影响其他内容。
- 用户取消批量修改公开或私有状态后，已完成结果应保留。
- 删除内容前必须二次确认。
- 导出前必须按权限过滤内容。
- 导出包含私有内容时必须二次确认。
- 过期导出记录不可继续下载。
- HTML 设定集导出模板是随系统发布的静态资源，不属于提示词模块或用户上传内容。
- HTML 设定集模板必须支持数据集元信息、目录、内容正文、离线打开、浏览器打印和 PDF 生成。
- 公开和私有状态、分享链接公开状态、静态展示页面发布状态相互独立，修改其中一个状态不得隐式修改另外两个状态。
- 公开和私有状态变更不得改变已创建分享链接的访问性。

### 分享规则

- 分享链接公开状态与内容平台内私有状态相互独立。
- 批量创建分享链接时，每条分享链接仍必须独立设置公开或私有。
- 批量创建分享链接必须展示成功数、失败数和失败原因，单条失败不得影响其他内容。
- 用户取消批量创建分享链接后，已创建成功的分享链接应保留。
- 公开分享私有内容必须二次确认，确认文案固定为："你正在公开分享平台内私有内容。创建后，获得链接的人无需登录即可查看这些内容。此操作不会改变内容在平台内的私有状态，但会绕过平台内列表、搜索和问答的可见性限制。是否继续？"
- 私有内容创建私有分享时，提示文案固定为："该分享包含平台内私有内容。私有分享仅具备分享访问权限的后台用户可访问，内容在平台内的私有状态不变。"
- 私有分享仅具备分享访问权限的后台用户可访问。
- 撤销或过期后链接不可访问。
- 分享目标被删除时显示占位，其余目标正常展示。

## Acceptance Criteria

- 用户能从三才图会门类进入卷，再进入具体条目。
- 用户能浏览 14 个正式门类、106 卷和卷首辅助内容。
- 用户能编辑三才图会条目并看到保存状态反馈。
- 用户能通过当前卷内搜索定位三才图会条目。
- 用户能按翻译、配图、视觉资产、完善和公开状态筛选三才图会条目。
- 用户能查看多张配图，并区分原图和视觉资产生成图。
- 用户能按原图、理解、融合、描述和生图流程完成视觉资产生产。
- 用户能调整融合权重。
- 用户能查看视觉资产历史产物并选择当前使用版本。
- 用户能发布草稿条目、归档已发布条目，并恢复已归档条目。
- 用户能新增王圻文档并维护正文。
- 用户能关联或替换王圻文档原始文件。
- 用户能在王圻文档详情中查看正文、摘要、标签和问答对。
- 用户能触发王圻文档摘要、标签和问答对生成，并在确认后应用。
- 用户能从王圻文档详情发起围绕当前文档的问答。
- 用户能在王圻文档列表浏览和时间线浏览之间切换。
- 用户能按标签筛选明代习俗条目。
- 用户能通过关键词搜索明代习俗条目。
- 用户能打开明代习俗详情并阅读安全渲染后的内容。
- 用户能在内容上下文中维护摘要、标签和问答对。
- 用户能查看和恢复历史版本。
- 权限不足用户看不到私有内容。
- 归档三才图会条目不出现在默认列表中。
- 用户能多选内容并批量修改公开或私有状态，完成后看到成功数、失败数和失败原因。
- 用户能选择范围和格式后生成导出产物。
- 用户能导出可离线打开的三才图会视觉资产设定集。
- 用户无权导出的内容不会进入导出结果。
- 过期导出记录不可继续下载。
- 生成的三才图会静态展示页面可独立打开并按门类、卷和条目逐层浏览。
- 用户能创建分享链接并复制。
- 用户能对多选内容批量创建分享链接。
- 批量创建分享链接完成后，用户能看到成功数、失败数和失败原因。
- 被撤销或过期的链接不可访问。
- 分享目标被删除时访问页显示占位提示。
- 私有分享未登录时引导登录。
