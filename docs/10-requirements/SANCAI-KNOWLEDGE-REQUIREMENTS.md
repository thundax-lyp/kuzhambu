# Sancai Knowledge Requirements

## Purpose

三才图会知识库模块定义《三才图会》内容浏览、整理和维护的需求，是平台核心内容需求。

## Scope

覆盖：
- 14 个正式门类、106 卷、卷首辅助内容和条目的三级浏览。
- 条目查看、创建、编辑、删除。
- 条目标题、门类、卷、原文、译文、标签、配图和状态展示。
- 摘要和问答对内联管理。
- 条目筛选、当前卷内搜索、分页和批量选择。
- 自动保存、手动保存和草稿恢复。
- 版本历史、版本对比和历史恢复。
- 草稿、发布和归档生命周期。
- 公开和私有可见性管理。
- CSV、JSON 和 HTML 设定集导出。
- 三才图会静态展示页面生成。
- 原图、图片理解结果、视觉描述和 AI 生成图组成的视觉资产生产。
- 视觉资产历史版本、当前使用版本和视觉资产设定集导出。

不覆盖：
- AI 翻译、标签提取和图片理解的执行。
- 跨库搜索和智能问答。
- 在线分享链接。
- CDN 分发。
- 图片裁剪、压缩、水印。
- 视频和音频处理。


## Functional Requirements

- 必须支持按门类、卷、条目逐级浏览。
- 必须支持 14 个正式门类、106 卷和卷首辅助内容浏览。
- 必须支持门类和卷的稳定排序。
- 必须支持条目基础 CRUD。
- 必须支持编辑条目标题、门类、卷、原文、译文和标签。
- 必须展示原文、译文、标签、配图和状态。
- 必须支持多张配图展示。
- 必须区分原始配图和视觉资产工作流生成图。
- 必须支持配图缩略预览和放大浏览。
- 必须支持从条目上下文进入该条目的视觉资产工作流。
- 必须支持原图上传、删除和预览。
- 必须支持基于图片理解结果进行信息融合。
- 必须支持文本和图片理解权重调节。
- 必须支持视觉描述生成、编辑和预览。
- 必须支持 AI 生图。
- 必须区分原图和 AI 生成图。
- 必须支持视觉资产产物历史和当前使用版本。
- 必须支持对多选条目批量执行视觉资产相关处理。
- 必须支持在条目编辑页或详情页内联查看、编辑和保存摘要。
- 必须支持在条目编辑页或详情页内联查看、编辑、删除和新增问答对。
- 必须支持分页、筛选、当前卷内搜索和多选。
- 分页必须至少支持每页 50、100 和 200 条。
- 必须支持按翻译状态、配图状态、视觉资产状态、完善状态和公开状态筛选。
- 必须支持对多选条目批量修改公开或私有状态。
- 必须支持自动保存和手动保存。
- 必须支持草稿恢复提示。
- 必须支持版本历史、版本对比和历史恢复。
- 必须支持草稿、发布和归档生命周期管理。
- 必须支持公开和私有可见性管理。
- 必须支持按门类、卷、筛选结果或选中条目导出 CSV、JSON 和 HTML 设定集。
- 必须支持按条目、筛选结果或选中视觉资产导出 HTML 视觉资产设定集。
- 必须支持对多选条目生成导出产物。
- 必须支持用户查看、下载和删除自己的导出记录。
- 必须展示导出产物的生成时间、条目数量和过期状态。
- 必须展示视觉资产设定集导出产物的生成时间、资产数量和过期状态。
- 必须支持三才图会静态展示页面生成。
- 必须支持静态展示页面按门类概览、卷列表和条目详情逐层浏览。
- 静态展示页面门类概览必须展示门类名称、条目总数、配图条目数和代表性缩略图。
- 静态展示页面条目详情必须展示条目名称、原文、译文、标签、配图和视觉资产描述。
- 必须支持静态展示页面搜索、筛选、浏览模式切换和响应式布局。

## Business Rules

- 自动保存用于防丢，不等同于正式版本。
- 手动保存、AI 结果应用和历史恢复应产生正式版本。
- 摘要和问答对属于内容上下文内的精修能力，不要求用户进入独立数据精修工作台。
- 私有条目只对授权用户可见。
- 新建条目应先进入草稿状态。
- 已发布条目不可退回草稿；需要退出默认可见范围时应归档。
- 草稿、发布和归档生命周期与公开或私有可见性相互独立。
- 批量修改公开或私有状态必须复用单条状态修改规则，并遵守当前用户权限。
- 批量修改公开或私有状态必须展示成功数、失败数和失败原因，单条失败不得影响其他条目。
- 用户取消批量修改公开或私有状态后，已完成结果应保留。
- 删除条目前必须二次确认。
- 导出前必须按权限过滤内容。
- 导出包含私有条目时必须二次确认。
- 过期导出记录不可继续下载。
- HTML 设定集导出模板是随系统发布的静态资源，不属于提示词模块或用户上传内容。
- HTML 设定集模板必须支持数据集元信息、目录、条目卡片、配图引用、离线打开、浏览器打印和 PDF 生成。
- 三才图会静态展示页面模板是随系统发布的静态资源，必须与 HTML 设定集模板分离。
- 静态展示页面默认只包含公开且未归档条目。
- 静态展示页面包含私有条目时必须由管理员确认，确认文案固定为："你正在将平台内私有内容写入三才图会静态展示页面。生成后的 HTML 文件将脱离平台登录、权限、撤销和搜索可见性控制；任何获得文件或访问地址的人都可能查看这些内容。此操作不会改变内容在平台内的私有状态。是否继续生成？"
- 静态展示页面一旦生成即脱离平台权限、撤销和内容状态变更自动控制。
- 公开和私有状态、分享链接公开状态、静态展示页面发布状态相互独立，修改其中一个状态不得隐式修改另外两个状态。
- 公开和私有状态变更不得改变已创建分享链接的访问性。
- 归档条目不参与默认列表、搜索、问答、导出和静态展示。
- 归档条目可在知识图谱中保留来源标记。
- 条目删除后，关联分享目标应显示内容已删除的占位提示，不应导致分享页崩溃。
- 条目拆分后保存的子条目是独立条目；父条目被删除时不应自动删除已保存子条目。
- 图片缺失不得影响正文阅读。
- 图片缺失时必须显示占位提示。
- 视觉资产必须关联明确三才图会条目。
- 原图和 AI 生成图必须可区分。
- 生图失败不得影响已有原图和历史产物。
- 删除原图或视觉资产产物前必须二次确认。
- 批量视觉资产处理必须复用单条视觉资产流程规则，单条失败不影响其他条目。
- 用户取消批量视觉资产处理后，已完成结果应保留。
- 视觉资产设定集导出模板是随系统发布的静态资源，不属于提示词模块或用户上传内容。
- 视觉资产设定集必须包含条目来源、视觉描述、色彩或风格信息、原图和 AI 生成图引用。
- 视觉资产设定集导出前必须按权限过滤关联条目和视觉资产。
- 用户可以选择当前使用版本。

## Acceptance Criteria

- 用户能从门类进入卷，再进入具体条目。
- 用户能浏览 14 个正式门类、106 卷和卷首辅助内容。
- 用户能编辑条目并看到保存状态反馈。
- 用户能通过当前卷内搜索定位条目。
- 用户能按翻译、配图、视觉资产、完善和公开状态筛选条目。
- 用户能查看多张配图，并区分原图和视觉资产生成图。
- 用户能按原图、理解、融合、描述和生图流程完成视觉资产生产。
- 用户能调整融合权重。
- 用户能查看视觉资产历史产物并选择当前使用版本。
- 用户能在条目上下文中维护摘要和问答对。
- 用户能恢复历史版本且恢复行为可追溯。
- 权限不足用户看不到私有条目。
- 归档条目不出现在默认列表中。
- 用户能发布草稿条目、归档已发布条目，并恢复已归档条目。
- 用户能多选条目并批量修改公开或私有状态。
- 批量修改公开或私有状态完成后，用户能看到成功数、失败数和失败原因。
- 用户能对多选条目发起批量视觉资产处理，并查看成功数、失败数和失败原因。
- 用户能选择三才图会范围和格式后生成导出产物。
- 用户能导出可离线打开的视觉资产设定集。
- 用户无权导出的条目不会进入导出结果。
- 过期导出记录不可继续下载。
- 生成的三才图会静态展示页面可独立打开。
- 生成的三才图会静态展示页面能按门类、卷和条目逐层浏览。
