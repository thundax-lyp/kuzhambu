# UI Rules

## Purpose

本文档定义「三才翰典」平台稳定的 UI 风格和交互原则。它用于约束界面设计方向，不替代具体页面设计、组件规范或视觉稿。

## Core Style

- 平台 UI 采用内容优先、学术极简、工具型体验。
- 界面应服务古籍原文、译文、标签、来源、图片和研究线索的阅读、整理、追溯和复用。
- UI 元素不得喧宾夺主；避免装饰性堆叠、营销式 hero、强视觉包装和无业务意义的氛围图形。
- 视觉气质应安静、克制、清晰、可长时间使用。
- 页面信息密度应适合研究、编辑和管理工作，避免把工作台做成展示官网。

## Layout Principles

- 平台内主界面以顶部栏、侧边栏和主内容区为基础。
- 顶部栏承载全局导航、当前位置、搜索和用户入口。
- 侧边栏承载知识库、工具和管理入口。
- 主内容区优先保障正文阅读、列表筛选、编辑和结果对比。
- 小屏界面应支持侧边栏折叠和单列阅读。
- 分享访问页、登录页、静态展示页面等特殊页面可以不使用平台内主布局，但仍应保持整体学术气质。

## Interaction Principles

- 高频工作流应减少点击层级，优先提供批量入口、内联编辑和清晰反馈。
- AI 生成结果必须先进入候选结果区，用户确认后才写入正式内容或产物。
- 长任务、AI 调用、批量操作、导出、备份和恢复必须展示清晰状态。
- 失败状态必须说明可理解原因，并在可重试时提供重试入口。
- 空状态应告诉用户当前没有什么，以及可执行的操作。
- 危险操作必须二次确认，确认文案应说明影响范围。

## Content Display

- 古籍原文、译文、摘要、标签、来源、图片和问答来源必须易读、易定位。
- 列表页应支持快速扫描，详情页应支持沉浸阅读和就地维护。
- 来源、权限状态、生命周期状态和 AI 生成状态应明确可见。
- 文本内容不得被视觉装饰、浮层或控件遮挡。
- Markdown 内容必须安全渲染，并保持阅读排版稳定。

## Visual Treatment

- 色彩应克制，不使用大面积高饱和、强渐变或装饰性光效作为主视觉。
- 字体选择应兼顾学术气质与屏幕可读性。
- 卡片、表格、列表、弹窗和工具栏应服务信息组织，不作为装饰层反复嵌套。
- 图标和按钮应表达明确操作；图标含义不明显时应提供可理解提示。
- 视觉资产、古籍配图和静态展示页面应突出真实内容，不使用与内容无关的占位式视觉包装。

## Accessibility

- 交互元素应支持键盘导航。
- 正文、按钮、标签和状态文本应保持足够对比度。
- 动效不得影响阅读和操作；必要时应支持减少动效。
- 响应式布局不得导致文本、控件或内容互相遮挡。

## Boundary

- 本文档只写稳定 UI 原则。
- 具体页面结构、组件尺寸、设计令牌、动效细节和页面模板应进入专项 UI 设计文档。
- 业务功能是否存在、权限规则和验收口径应进入对应 `docs/10-requirements/*-REQUIREMENTS.md`。
- 只为已确认页面、组件和功能编写 UI 规则。
